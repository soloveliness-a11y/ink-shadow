# 剧本杀游戏系统 - 质量检查流程

## 概述

本流程用于确保剧本适配质量，保证剧本能在系统中正常运行。

## 检查阶段

### 阶段1：JSON格式检查

**检查内容**：
- 所有JSON文件格式正确
- 必填字段完整
- 字段名称符合schema定义

**检查方法**：
```bash
cd /Users/a1np/文档/Design/murder-mystery-game
for f in content/<剧本名>/*.json content/<剧本名>/characters/*.json; do
  python3 -m json.tool "$f" > /dev/null && echo "✓ $f" || echo "✗ $f"
done
```

**通过标准**：所有JSON文件验证通过

### 阶段2：Schema验证

**检查内容**：
- JSON结构符合schema定义
- 字段类型正确
- 引用关系有效

**检查方法**：
```bash
cd /Users/a1np/文档/Design/murder-mystery-game
npx ts-node -e "
import { validateScript } from './packages/schema/src/validate';
import * as fs from 'fs';
const script = JSON.parse(fs.readFileSync('content/<剧本名>/meta.json', 'utf-8'));
const result = validateScript(script);
console.log(result.errors.length === 0 ? '✓ 验证通过' : '✗ 验证失败');
result.errors.forEach(e => console.log('  -', e));
"
```

**通过标准**：无验证错误

### 阶段3：机制兼容性检查

**检查内容**：
- 系统支持剧本使用的所有机制
- 前端UI能正确显示
- 后端能正确处理

**检查方法**：
1. 检查剧本使用的机制列表
2. 对照系统能力清单
3. 验证前端UI支持
4. 验证后端处理逻辑

**通过标准**：所有机制系统都支持

### 阶段4：功能测试

**检查内容**：
- 剧本能正常加载
- 阶段能正确推进
- 线索能正常搜查
- 投票能正常进行
- 结局能正确显示

**检查方法**：
1. 启动本地服务器
2. 创建测试房间
3. 选择剧本
4. 模拟玩家操作
5. 验证所有功能

**通过标准**：所有功能正常运行

### 阶段5：兼容性测试

**检查内容**：
- 与其他剧本兼容
- 系统升级后仍能运行
- 不同浏览器/设备兼容

**检查方法**：
1. 选择多个剧本进行测试
2. 升级系统后重新测试
3. 在不同浏览器/设备上测试

**通过标准**：所有测试通过

## 问题处理

### 问题1：JSON格式错误
**处理方法**：
1. 检查错误信息
2. 修正JSON格式
3. 重新验证

### 问题2：Schema验证失败
**处理方法**：
1. 检查验证错误
2. 修正字段定义
3. 重新验证

### 问题3：机制不支持
**处理方法**：
1. 检查系统能力清单
2. 确认是否已实现
3. 如未实现，记录需求

### 问题4：功能测试失败
**处理方法**：
1. 检查错误日志
2. 定位问题原因
3. 修复问题
4. 重新测试
